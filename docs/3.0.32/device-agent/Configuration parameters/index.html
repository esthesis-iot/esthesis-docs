Configuration parameters

The esthesis device agent supports a plethora of rutime configuration parameters to match your environment and runtime needs. Parameters can be defined either as environment variables or as command line arguments. The following tables lists all available configuration parameters.

Mandatory parameters
Environment/CLI variableDescriptionType
HARDWARE_ID, hardwareIdThe unique identifier of the device.Alphanumeric
REGISTRATION_URL, registrationUrlThe URL of esthesis server to register this device with.URL
Optional parameters
Environment/CLI variableDescriptionType
ATTRIBUTES, attributesA comma-separated list of key-value pairs to be sent as attributes.Alphanumeric
AUTO_UPDATE, autoUpdateA flag indicating whether the device should try to automatically obtain newer firmware once per day.

Default: false		Boolean
DEMO_CATEGORY, demoCategoryThe category of data posted as demo data.

Default: demo		Alphanumeric
DEMO_INTERVAL, demoIntervalThe frequency in which demo data is generated (in seconds).

Default: 0		Numeric
ENDPOINT_HTTP, endpointHttpWhether the embedded HTTP server is enabled or not.

Default: false		Boolean
ENDPOINT_HTTP_LISTENING_IP, endpointHttpListeningIPThe IP address where the embedded HTTP server listens to.

Default: 127.0.0.1		IP address
ENDPOINT_HTTP_LISTENING_PORT, endpointHttpListeningPortThe port in which the embedded HTTP server listens to.

Default: 8080		Numeric
ENDPOINT_MQTT, endpointMqttWhether the embedded MQTT server is enabled or not.

Default: false		Boolean
ENDPOINT_MQTT_LISTENING_IP, endpointMqttListeningIPThe IP address where the embedded MQTT server listens to.

Default: 127.0.0.1		IP address
ENDPOINT_MQTT_LISTENING_PORT, endpointMqttListeningPortThe port in which the embedded MQTT server listens to.

Default: 1883		Numeric
HEALTH_REPORT_INTERVAL, healthReportIntervalThe frequency to send health reports (in seconds).

Default: 300		Numeric
HTTP_RETRY, httpRetryThe number of seconds to wait before retrying a failed HTTP request.

Default: 60		Numeric
HTTP_TIMEOUT, httpTimeoutThe number of seconds after which an HTTP call times out.

Default: 60		Numeric
LOG_ABBREVIATION, logAbbreviationThe characters length to abbreviate log messages to.

Default: 1024		Numeric
LOG_LEVEL, logLevelThe logging level to use [trace, debug, info].

Default: info		Alphanumeric
LUA_HTTP_TELEMETRY_SCRIPT, luaHttpTelemetryScriptThe filesystem location of a Lua script to transform incoming payloads for telemetry data pushed via the HTTP endpointAlphanumeric
LUA_HTTP_METADATA_SCRIPT, luaHttpMetadataScriptThe filesystem location of a Lua script to transform incoming payloads for metadata data pushed via the HTTP endpointAlphanumeric
LUA_MQTT_TELEMETRY_SCRIPT, luaMqttTelemetryScriptThe filesystem location of a Lua script to transform incoming payloads for telemetry data pushed via the MQTT endpointAlphanumeric
LUA_MQTT_METADATA_SCRIPT, luaMqttMetadataScriptThe filesystem location of a Lua script to transform incoming payloads for metadata data pushed via the MQTT endpointAlphanumeric
-, luaExtraMqttTelemetryTopicA custom MQTT telemetry endpoint to be handled by a user-defined Lua script. The first argument of this parameter is the name of the topic, and the second argument is the Lua script to be executed. This parameter can be repeated to define additional topics.Alphanumeric Alphanumeric
-, luaExtraMqttMetadataTopicA custom MQTT metadata endpoint to be handled by a user-defined Lua script. The first argument of this parameter is the name of the topic, and the second argument is the Lua script to be executed. This parameter can be repeated to define additional topics.Alphanumeric Alphanumeric
-, luaExtraHttpTelemetryEndpointA custom HTTP telemetry endpoint to be handled by a user-defined Lua script. The first argument of this parameter is the name of the endpoint, and the second argument is the Lua script to be executed. This parameter can be repeated to define additional topics.Alphanumeric Alphanumeric
-, luaExtraHttpMetadataEndpointA custom HTTP metadata endpoint to be handled by a user-defined Lua script. The first argument of this parameter is the name of the endpoint, and the second argument is the Lua script to be executed. This parameter can be repeated to define additional topics.Alphanumeric Alphanumeric
MQTT_TIMEOUT, mqttTimeoutThe number of seconds to wait before failing an outgoing MQTT message.

Default: 60		Numeric
PAUSE_STARTUP, pauseStartupA flag indicating whether the device should start paused.

Default: false		Boolean
PING_INTERVAL, pingIntervalThe frequency to ping esthesis Core (in seconds).

Default: 60		Numeric
PROPERTIES_FILE, propertiesFileThe file to store the agentâ€™s configuration.

Default: $HOME/.esthesis/device/esthesis.properties		Alphanumeric
PROVISIONING_SCRIPT, provisioningScriptThe script used to install new provisioning packages.

Default: $HOME/.esthesis/device/firmware.sh		Alphanumeric
REBOOT_SCRIPT, rebootScriptThe script used to reboot the device.

Default: $HOME/.esthesis/device/reboot.sh		Alphanumeric
REGISTRATION_SECRET, registrationSecretIf set, the registration request will include it as a header.Alphanumeric
SECURE_PROPERTIES_FILE, securePropertiesFileThe secure file to store sensitive parts of the agent's configuration.

Default: $HOME/.esthesis/device/secure/esthesis.properties		Alphanumeric
SECURE_PROVISIONING, secureProvisioningA flag indicating whether provisioning requests should be accompanied by a signature token.

Default: false		Boolean
SHUTDOWN_SCRIPT, shutdownScriptThe script used to shut down the device.

Default: $HOME/.esthesis/device/shutdown.sh		Alphanumeric
SIGNATURE_ALGORITHM, signatureAlgorithmThe algorithm to use to produce signatures.

Default: SHA256WITHRSA		Alphanumeric
SUPPORTED_COMMANDS, supportedCommandsThe remote commands this device supports:
e: Execute arbitrary
f: Firmware update
r: Reboot
s: Shutdown
p: Ping
h: Health report

Default: efrsph		Alphanumeric
TAGS, tagsA comma-separated list of tags to associate with this device.Alphanumeric
TEMP_DIR, tempDirThe folder to temporarily store provisioning packages.

Default: OS default temp directory		Alphanumeric
TOPIC_COMMAND_REQUEST, topicCommandRequestThe MQTT topic to use for command request messages.

Default: esthesis/command/request		Alphanumeric
TOPIC_COMMAND_REPLY, topicCommandReplyThe MQTT topic to use for command reply messages.

Default: esthesis/command/reply		Alphanumeric
TOPIC_DEMO, topicDemoThe MQTT topic to post demo data.

Default: esthesis/telemetry		Alphanumeric
TOPIC_METADATA, topicMetadataThe MQTT topic to use for metadata messages.

Default: esthesis/metadata		Alphanumeric
TOPIC_PING, topicPingThe MQTT topic to use for ping messages.

Default: esthesis/ping		Alphanumeric
TOPIC_TELEMETRY, topicTelemetryThe MQTT topic to use for telemetry messages.

Default: esthesis/telemetry		Alphanumeric
VERSION_FILE, versionFileA file with a single line of text depicting the current version of the firmware running on the device.

Default: $HOME/.esthesis/device/version		Alphanumeric
VERSION_REPORT_TOPIC, versionReportTopicThe MQTT topic to report the firmware version.

Default: esthesis/metadata		Alphanumeric
VERSION_REPORT, versionReportReport the version number available in the specified version file once during boot.

Default: false		Boolean
